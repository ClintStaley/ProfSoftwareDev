Code Modification/Debugging

Comprehend the Code

Before you touch a line of code, be sure you comprehend it as well as its author did. 90% of the value of code is in the mind of the developer, and unless you rebuild that 90% in your own comprehension, you'll make mistakes and degrade the code. And don't settle for a "general impression" of how the code works; be precise. You should be able to explain every operator, variable, etc.

Be Consistent

A big codebase is highly complex. Consistency is essential. This applies to variable and function naming, patterns of function use, coding style, etc. You need an almost obsessive concern for order to fight the entropy of software changes.

Keep Code General

Design parameter lists, classes, functions, etc. for general use, not just for the immediate specific use you are putting them to.

Flow

You are most productive when you're in a coding "flow". This happens when you've come up to speed on the code and are incrementally adding/fixing code at a steady pace. It's generally also when coding is the most fun. Avoid breaking flow if you can. Work in long enough stints to get into a flow and stay there.
